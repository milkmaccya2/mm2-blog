# テスト自動化導入計画

## ゴール
ブログの品質を維持し、将来的なリファクタリングや機能追加時の安全性を確保するために、E2Eテスト（Playwright）を導入します。
ユニットテスト（Vitest）については、現状複雑なロジックが少ないため、今回は導入を見送りますが、必要に応じて追加できるように構成します。

## ユーザーレビューが必要な項目
- **Playwrightの採用**: ブラウザベースのE2EテストツールとしてPlaywrightを採用します。
- **CI/CD統合**: GitHub Actionsで `npm run build` 後にテストを実行するように設定します。

## 変更内容

### テスト環境の構築
#### [NEW] Playwright設定ファイル
- `playwright.config.ts`: Playwrightの設定（ブラウザ、ベースURLなど）

#### [NEW] テストコード
- `tests/e2e.spec.ts`: 基本的なページ遷移と表示確認
  - トップページの表示確認
  - 記事一覧の表示確認
  - 記事詳細ページへの遷移確認
  - 404ページの確認（存在しないURLへのアクセス）

### CI/CD設定
#### [MODIFY] .github/workflows/ci.yml
- `test` ジョブの追加
- Playwrightのインストールとテスト実行ステップの追加

## 検証計画

### 自動テスト
- ローカル環境で `npx playwright test` を実行し、全テストがパスすることを確認します。
- GitHub Actions上でワークフローが正常に動作し、テストがパスすることを確認します。

### 手動確認
- 生成されたテストレポート（HTML）を確認し、内容が意図通りであることを確認します。
