# 実装計画：未使用ファイルの整理

## 目的
プロジェクト内の未使用ファイルを特定し、整理（削除）することで、メンテナンス性を向上させる。

## ユーザーレビューが必要な事項
- 削除候補ファイルのリスト（削除前に必ず確認を行う）

## 提案する変更
### 調査手順
1. `src` ディレクトリ以下の全ファイルを取得する。
2. 各ファイルについて、プロジェクト内での参照状況を確認する。
   - `pages` ディレクトリ内のファイルはルート定義として扱われるため、直接の参照がなくても使用されているとみなす（ただし、動的ルートなどの兼ね合いで注意が必要）。
   - `components`, `layouts`, `assets`, `styles` 内のファイルは、他のファイルからインポートされているかを確認する。
3. `public` ディレクトリ内のファイルも同様に参照を確認する。

### 削除候補の特定基準
- 他のファイルから `import` されていない、または `url()` などで参照されていないファイル。
- `pages` ディレクトリ以外にある孤立したファイル。

### 削除候補
以下のファイルはプロジェクト内で参照されておらず、削除可能と判断されました。

#### 画像ファイル (未使用)
- `src/assets/blog-placeholder-2.jpg`
- `src/assets/blog-placeholder-4.jpg`
- `src/assets/blog-placeholder-5.jpg`

#### 設定ファイル (重複)
- `src/content/config.ts`
  - 現在 `src/content.config.ts` (Astro v5推奨) と重複しています。
  - `config.ts` にのみ存在する定義 (`type`) がありますが、記事ファイルで使用されていない場合は `content.config.ts` を残してこちらは削除します。



### 実行計画
1. `src/content.config.ts` に必要な定義があればマージ（必要なければスキップ）。
2. 上記候補ファイルを削除。
3. `npm run build` で影響確認。


## 検証計画
### 自動テスト
- `npm run build` を実行し、削除によってビルドエラーが発生しないことを確認する。

### 手動確認
- 主要なページが表示されることを確認する。
