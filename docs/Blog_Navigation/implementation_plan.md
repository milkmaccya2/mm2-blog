# 実装計画: ブログ記事のナビゲーション追加

## 1. ページロジックの更新 (`src/pages/blog/[...slug].astro`)
- `getStaticPaths` 関数内でブログ記事を取得し、日付順（降順）にソートする。
- 各記事に対して、前の記事 (`prevPost`) と次の記事 (`nextPost`) を特定する。
  - `prevPost`: 配列内の次の要素（もし存在すれば）。インデックス + 1。
  - `nextPost`: 配列内の前の要素（もし存在すれば）。インデックス - 1。
- 特定した `prevPost` と `nextPost` を `props` として `BlogPost` レイアウトコンポーネントに渡す。

## 2. ナビゲーションコンポーネントの作成 (`src/components/BlogNavigation.astro`)
- 共通化のため、ナビゲーションロジックを分離したコンポーネントを作成する。
- **要件**:
  - `prevPost` と `nextPost` をプロパティとして受け取る。
  - **レイアウト**:
    - モバイル・デスクトップ共に **横並び (Flexbox)** を維持する。
    - モバイルではスペースが限られるため、文言を短縮（例：「前の記事」→「前へ」）し、非表示クラス (`hidden md:inline`) を活用する。
    - タイトルは引き続き省略表示 (`truncate`) を行う。

## 3. レイアウトの更新 (`src/layouts/BlogPost.astro`)
- `BlogPost` レイアウト内で `BlogNavigation` コンポーネントをインポートする。
- ナビゲーションを以下の2箇所に配置する：
  1. **ページ上部**: タイトル下・目次（TOC）の上。
  2. **ページ下部**: コンテンツ (`<slot />`) の下。
