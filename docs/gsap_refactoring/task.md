# GSAP アニメーション共通化リファクタリング計画

## 目的
現状 `index.astro` にハードコーディングされている GSAP アニメーションのロジックを共通モジュール化し、他のページ（ブログ記事詳細や About ページなど）でもクラス名を付与するだけでアニメーションを利用できるようにする。また、ページ遷移時のクリーンアップやエラー防止処理を一元管理する。

## タスク
- [ ] `src/scripts/animations.ts` の作成
    - GSAP および ScrollTrigger の登録
    - `astro:page-load` イベントでのアニメーション初期化処理の実装
    - `astro:before-swap` イベントでのクリーンアップ処理の実装
    - 共通アニメーション（`.fade-in-up`, `.reveal-wrap` 等）の定義
- [ ] `src/layouts/BaseLayout.astro` の修正
    - 作成した `src/scripts/animations.ts` を読み込む設定を追加
- [ ] `src/pages/index.astro` の修正
    - ページ内のインラインスクリプトを削除し、共通スクリプトに依存するように変更
    - ページ固有のアニメーション設定があれば、必要に応じて分離または統合を検討
- [ ] 動作確認
    - トップページでアニメーションが維持されていることの確認
    - 他のページに遷移してもコンソールエラー（警告）が出ないことの確認
